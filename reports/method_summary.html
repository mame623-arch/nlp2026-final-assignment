1. Method

오류 분석에서 줄일 수 있는 오류(C)가 scope·lexical_gap에 집중된다는 진단으로부터, 각 오류 유형을 겨냥한 GPT-2[1] 기반 멤버를 만들고 스태커로 결합한다. 다양성이 시드·데이터가 아니라 아키텍처에서 온다.

① cloze — 집계(전반)에 강함

문장쌍 (s1,s2)
cloze 프롬프트로 결합
→
GPT-2
마지막 토큰 표현
→

핵심: PET류 cloze-verbalizer[2] 방식 — 두 문장을 한 시퀀스로 합쳐 cross-attention으로 읽어 전반적 의미 일치를 판단. 단, 비교를 마지막 토큰 하나로 압축해 "무엇이 안 겹치는지(scope)"는 묻힌다 → 그래서 ②③이 필요.

② alignment (ESIM) — scope 겨냥

s1, s2 별도 인코딩
GPT-2 → 토큰표현 H1,H2
→
BiLSTM
문맥 재인코딩
→
soft-alignment
(cross-attention)
각 토큰이 상대 문장의 어느 토큰과 대응되는지
→
enhance·compose
[P, P̃, P−P̃, P⊙P̃] → BiLSTM → pool
→
head → margin
핵심: ESIM[3]·Decomposable Attention[4]의 정렬 메커니즘을 채택. cross-attention으로 매칭되지 않는 토큰을 찾아낸다. 한 문장에만 있는 절은 대응 토큰이 없어 정렬 잔차 P−P̃가 커진다 → 한쪽에만 있는 내용(scope)을 표면 단어가 아니라 정렬 구조로 포착. paraphrase = 양방향 정렬이 모두 채워질 때.

③ bi-enc — lexical_gap 겨냥 (alignment의 step1 = 정렬 없는 대조군)

s1, s2 별도 인코딩
GPT-2 → pool(mean+max+last)
→
문장 임베딩 u, v
각 문장을 1개 벡터로
→
상호작용 피처
[u, v, |u−v|, u⊙v]
→
head → margin
핵심: InferSent[5]·Sentence-BERT[6]의 bi-encoder 상호작용 피처. 토큰 정렬 없이 문장 전체를 하나의 의미 벡터로 압축해 비교 → 표면 단어가 달라도 임베딩 거리 |u−v|가 작으면 같은 의미로 판단해 lexical_gap을 교정. (대조 학습 SimCSE[7]로 표면불변성 강화 가능)

④ cloze+NLI — negation·entity·OOD 견고성

+

양방향 함의/모순 확률
+

핵심: 외부 사전학습 NLI — RoBERTa[13]를 MNLI[12]로 미세조정한 모델 + MiniLM[14] 임베딩 — 의 함의/모순 지식을 빌려 cloze 보강. 부정·반의(negation)나 OOD처럼 quora만으로 안 잡히는 패턴을 떠받친다. (GPT-2 학습 없이 추론-time 신호로만)

⑤ 스태커 (ours, 앙상블 결합)

구조 멤버 margin ×3
cloze · alignment · bi-enc
+
외부 5피처
e_ab, e_ba, c_ab, c_ba, cos
→
z-score 표준화
→
Logistic Regression
2-fold cross-fit
→
yes / no
멤버 margin각 멤버의 yes − no logit 한 값(중복 확신도). ours-A는 cloze·alignment·bi-enc 3개 margin만.
외부 5피처e_ab=s1→s2 함의확률, e_ba=s2→s1 함의확률(둘 다 높으면 상호함의), c_ab/c_ba=양방향 모순확률(부정 신호), cos=MiniLM 코사인. ours-B는 3 margin + 이 5피처.
2-fold cross-fit스태커도 학습이 필요한데 같은 데이터로 또 학습하면 leakage. 데이터를 절반으로 갈라 fold1 라벨→fold2 예측, fold2 라벨→fold1 예측 → 모든 샘플을 자기 라벨 안 본 스태커로 평가. Stacked Generalization[8]의 표준 절차.
Ablationours-A: 구조 3 margin (순수 quora-only) · ours-B: + 외부 NLI
설계 논지: 각 멤버가 서로 다른 오류 유형을 1등으로 잡으면(상보성), 단일 모델이 못 잡는 오류를 스태커가 상보적으로 회수. 스태킹은 진단된 다양성의 필연적 귀결.

2.9 통계적 유의성 · 재현성 (신규)

seed 반복학습 단계에서 3 seed로 반복(초기화·셔플·dropout이 seed 의존) → 모델 3개 → 각 모델을 같은 test셋에서 평가 → 결과 칸은 평균 ± std. 평가(추론)는 결정적이라 seed 무관. 비용상 baseline·ours만 3 seed, 비교메소드는 1 seed로 축소 가능.
유의성 검정McNemar test[20] — ours vs baseline의 예측이 같은 테스트셋에서 유의하게 다른지(paired). 특히 MRPC(N=1725)처럼 작은 셋에서 필수.
신뢰구간metric별 bootstrap 95% CI(예: 1000 resample) → 표/그림에 오차막대.
보고 방식결과표는 평균±std, 본문엔 "ours−baseline Δ와 McNemar p, CI 비중첩" 명시.

3.2 A / B / C 정의

판정의미성격
A — 라벨 오류gold가 명백히 틀림 (정답이 반대)비가역 — 벤치마크 라벨 노이즈[17]
B — 모호사람도 합리적으로 불일치, 단일 정답 부적절비가역 — human label variation[15][16]
C — 모델 오류라벨은 맞는데 모델이 틀림 (judge의 OK)학습 가능 = 줄일 헤드룸
cleanAcc (clean accuracy): 전체 정확도에서 A·B를 제외하고 계산한 정확도 = "모델이 공정하게 책임질 케이스에서의 정확도". 비가역 노이즈를 걷어낸 실질 상한 추정. 줄일 여지(C) = 1 − cleanAcc.

tag정의예시 (gold)

4. 보고서 스토리라인

전체 흐름: GPT-2 cloze가 QQP ~90%에서 정체 → "왜?"를 오류분석으로 진단(비가역 A·B vs 학습가능 C, C는 scope·lexical에 집중) → 진단에 맞춘 표적 구조를 GPT-2 위에 추가 → 같은 오류분석 재실행으로 "그 오류가 줄었음" 입증 + 천장을 정직하게 분해.

3대 기여 — C1 천장 분해(LLM-judge+κ) · C2 진단→구조 연역 · C3 표적 오류 회수 closed-loop.

4.1 Introduction (6문단)

¶내용 · 인용
¶1배경+퍼즐 — paraphrase 정의·활용, GPT-2 cloze[1][2]가 강한 baseline이나 QQP[11] ~90% 정체. 태스크 자체 노이즈 큼[15][16] → "이 90%는 천장인가, 못 푼 것인가?"
¶2기존 한계 — 집계 정확도만 추구, 잔차 오류 균질 취급, 왜 막히는지 진단 안 함, 구조 선택이 오류에서 연역되지 않음
¶3아이디어① 진단 — 학습가능 오류가 서로 다른 유형(scope·lexical)에 집중, 요구 귀납편향 다름 → 단일 구조로 둘 다 못함
¶4아이디어② 구조+앙상블(미리보기) — 유형별 맞춤 구조를 GPT-2 위에, 스태킹[8]은 다양성의 필연. 차별: 표적 회수 검증(closed loop)+정직한 한계
¶5기여 정리 — C1 / C2 / C3
¶6논문 구성 로드맵

원칙: Intro엔 method 미리보기만, 본격 method(§4)는 진단(§3) 다음에.

4.2 Related Work (4문단, 각 "기존 vs 본 연구"로 마무리)

¶주제 · 차별점 · 인용
¶1벤치마크와 라벨 모호성 — QQP[11]·PAWS[9]·MRPC[10], label error[17]·human disagreement[15][16]. 차별: 노이즈를 천장의 정체로 정량 분해
¶2LM 기반 분류(cloze) — PET/cloze-verbalizer[2], GPT-2[1]. 차별: cloze를 진단 대상 baseline으로
¶3문장쌍 모델링 — bi-encoder(InferSent[5]·SBERT[6]), 정렬(ESIM[3]·Decomposable Attn[4]), 대조학습(SimCSE[7]). 차별: causal GPT-2 위에 from-scratch
¶4앙상블·스태킹 — stacked generalization[8]. 차별: 시드·데이터가 아니라 아키텍처 다양성

4.3 Conclusion (1문단)

오류 분석으로 QQP 정체의 천장을 비가역(라벨오류·모호) vs 학습가능(scope·lexical)으로 분해하고, 이 진단으로부터 정렬(scope)·bi-encoder(lexical)·cloze(집계) 멤버를 GPT-2 위에 연역해 스태킹했다. 같은 오류 분석을 재실행해 표적 오류가 실제로 줄었음을 closed-loop으로 입증했으며, 동일 quora-only 학습 하에서 ours가 ID(Quora)뿐 아니라 OOD(PAWS·MRPC)에서도 baseline을 일관되게 능가함을 보였다. (수치는 재실험 후 확정)

4.4 Limitation (1문단)

GPT-2는 causal 인코더라 양방향을 가정하는 ESIM 정렬과 구조적 미스매치가 있고, bi-encoder는 cloze의 cross-attention 일부를 잃는다. OOD 견고성의 상당 부분은 외부 사전학습 NLI(roberta-mnli) 신호에 의존하므로 "quora-only 학습 구조만의" 일반화와 외부 지식 기여를 ours-A/B ablation으로 구분해 보고한다. 또한 천장 분해의 판정자가 LLM이라 "LLM으로 LLM 한계를 잰다"는 순환성이 있어 human κ로 일부만 방어되며, MRPC dev(N=408)는 작아 통계적 불확실성이 크고, 단일 데이터 split·소수 seed에 의존한다. 따라서 본 연구의 기여는 절대 정확도보다 진단→설계→표적 회수의 인과 증명(C3)과 천장의 정직한 분해에 있다.
